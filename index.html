Street Leaf API's Using Azure Function Apps

Here we are using azure function app and triggers for create apis for data.

  • Version: 1.0
  • Created: 2 September, 2022

If you have any questions that are beyond the scope of this help file, Please feel free to email via Orange Data tech Support.

API's & URL's

Follow all are our apis which is associate with azure functions triggers:

  1. Start apis with base url currently here our base url https://streetleaf.azurewebsites.net after the base url we are using api prefix .
  2. after base_url & prefix we are using AZURE Function trigger name for example StreetleafLogin this trigger for login users :
    • base_url/api/trigger - Contains all of the assets referenced
      • base_url/api/Login - Login Users
      • base_url/api/GetProjectsList - Get Project List
      • base_url/api/GetProjectsDetails - Project Details
      • base_url/api/GetPolesList - Pole List
      • base_url/api/GetPoleModel – Single Pole Model.
      • base_url/api/GetCurrentReadings - All Current Readings
      • base_url/api/GetPolesDetails – Poles Details.
    • base_url/api - Homepage

Function App

Azure Function is a serverless compute service that enables user to run event-triggered code without having to provision or manage infrastructure. Being as a trigger-based service, it runs a script or piece of code in response to a variety of events. Azure Function App

Server Less API's

SAM is a tool for creating, updating, and managing serverless applications and all the resources needed for the application to operate. AWS SAM lets you automate the process of creating required services for your applications instead of creating them one by one on AWS console.
